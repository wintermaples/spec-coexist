# 基本設計書 サンプル・テンプレート調査まとめ

日本国内で高く評価されている基本設計書（外部設計書）のサンプル・テンプレート・ガイドを調査した結果をまとめる。

## 1. IPA（独立行政法人 情報処理推進機構）

### 1-1. 機能要件の合意形成ガイド
- 発行: IPA
- 内容: 要件定義→基本設計へ橋渡しする合意形成の手順と、基本設計書で押さえるべき項目（画面・帳票・バッチ・データモデル等）の記述方法を解説。
- 評価: 基本設計書の「何を書くべきか」の一次リファレンスとして最も評価が高い。

### 1-2. IPA 汎用的教育コンテンツ（設計書テンプレート）
- 内容: 基本設計書・詳細設計書の雛形を無料公開。データフロー図、ER図、画面レイアウト、項目定義などを含む。
- 特長: 教育用のため章立て・記述例が明瞭で、実務テンプレートのベースとして広く流用されている。

## 2. デジタル庁『デジタル・ガバメント推進標準ガイドライン 実践ガイドブック』
- 政府情報システムの基本設計書の標準構成（業務設計／アプリ設計／データ設計／インフラ設計／移行設計／運用設計／セキュリティ設計）を規定。
- 官公庁案件のデファクトで、民間の大規模案件の参考にもされる。

## 4. 基本設計書の典型的な章立て（上記資料の共通項）
1. はじめに（目的・適用範囲・用語定義）
2. システム全体概要／システム構成図
3. 業務フロー設計
4. 機能一覧／機能設計
5. 画面設計（画面一覧・遷移図・レイアウト・項目定義）
6. 帳票設計
7. 外部インターフェース設計
8. バッチ設計
9. データベース設計（ER図・テーブル定義・CRUD図）
10. 非機能設計（性能・可用性・セキュリティ・運用）
11. 移行設計
12. テスト方針

## 結論（最も評価されているサンプル）
1. **IPA『機能要件の合意形成ガイド』** が国内で最も権威ある一次資料。
2. **デジタル庁 標準ガイドライン実践ガイドブック** が政府系デファクト。
3. 実務雛形としては **若手エンジニアの羅針盤** と **NotePM** のテンプレートが民間で最も引用されている。
