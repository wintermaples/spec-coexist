# 要件定義書 サンプル・テンプレート調査まとめ

日本国内で高く評価されている要件定義書のサンプル・テンプレート・ガイドを調査した結果をまとめる。

## 1. IPA（独立行政法人 情報処理推進機構）

公的機関が発行しており、日本のSIer・ユーザー企業の実務で最も広く参照されている一次資料。実務者・書籍・各種ブログでも「まずはIPA」と推奨されることが多い。

### 1-1. 超上流から攻めるIT化の事例集：要件定義
- URL: https://www.ipa.go.jp/archive/digital/tools/ep/ep2.html
- 内容: 機能要件（プロセス／データ／インターフェース）および非機能要件について、業務流れ図・データ項目定義書・画面一覧などの実サンプルが収録されている。
- 特長: 実際の企業事例ベースなので、雛形だけでなく「どう書くか」の判断材料になる。

### 1-2. ユーザのための要件定義ガイド 第2版 ― 要件定義を成功に導く128の勘どころ
- PDF: https://www.ipa.go.jp/archive/publish/qv6pgp0000000wrt-att/000079352.pdf
- 内容: 発注者（ユーザー企業）視点で要件定義の進め方を128のポイントに整理。
- 評価: 初学者〜中堅PM/BAの定番教科書として最も高く評価されている。

### 1-3. 機能要件の合意形成ガイド
- IPA アーカイブ内で公開。基本設計フェーズとの接続を意識した要件合意の進め方を解説。

## 2. デジタル庁『デジタル・ガバメント推進標準ガイドライン 実践ガイドブック』
- 発行: デジタル庁
- 内容: 政府情報システムの調達で用いる要件定義書／基本設計書の標準構成を規定。
- 評価: 官公庁案件の事実上の標準。民間でも参考にされる。

## 3. 実案件の公開要件定義書（官公庁調達資料）
実物サンプルとして頻繁に引用される。
- 国土交通省『建設キャリアアップシステム 要件定義書』
- 厚生労働省 各システム調達 要件定義書
  → 業務要件・機能要件・非機能要件が一式揃っており、章立ての参考に最適。

## 5. 非機能要件のデファクト
- **IPA『非機能要求グレード』**: 非機能要件定義の国内デファクト。要件定義書の非機能章で必ず参照される。

## 結論（最も評価されているサンプル）
1. **IPA 超上流事例集** と **ユーザのための要件定義ガイド（128の勘どころ）** が総合評価で最上位。
2. 非機能要件は **IPA 非機能要求グレード** を併用するのが定番。
3. 官公庁調達の公開要件定義書（国交省・厚労省など）が実サンプルとして最良。
